# 汉画像石数字化研究平台 —— 完整开发需求文档（v3）

> 本文档供 AI IDE 工具直接读取执行。项目为基于 Web 浏览器的汉画像石数字化研究平台，核心定位是**三维浏览器 + 文物拼接复原工具**，并在后期扩展**专业标注系统**。技术栈由你（AI）根据项目特点自主选型，并在交付前给出选型说明。

## 📌 开发优先级

本项目按**三维浏览器**的产品定位推进，开发顺序为：

1. **第一优先级（P1）**：三维浏览模块（基础能力，必须最先建立）
2. **第二优先级（P2）**：多石拼接模块（核心差异化功能）
3. **第三优先级（P3）**：图像标注模块（学术研究扩展功能，最后开发）

> 理由：浏览与拼接是"工具骨架"，能快速形成可用产品；标注系统功能复杂（涉及 AI 集成、知识图谱、版本管理），适合在骨架稳定后独立展开。

# 🥇 第一部分：三维模型浏览模块（P1，最先开发）

## 🎯 功能目标

实现汉画像石三维模型的高性能加载与交互浏览，支持 3D / 2D 视图切换，作为整个平台的浏览基座。

## ✅ 实现步骤

### Step 1.1 — 后端模型服务

- 搭建后端服务（技术自选，推荐 Node.js + Express 或 Python + FastAPI）。
- 扫描 `E:\WSC3D\temp`，建立模型文件索引，对外提供 REST API：
  - `GET /api/stones` — 返回所有画像石列表（ID、名称、缩略图）。
  - `GET /api/stones/:id/model` — 返回指定画像石的模型文件。
  - `GET /api/stones/:id/metadata` — 返回解析后的结构化 JSON 数据。
- 静态文件托管模型资源，支持分块传输与缓存。

### Step 1.2 — 模型加载与渲染优化

- 前端使用 **Three.js**（推荐，生态完善）搭建 3D 场景。
- **性能优化策略**（必须实现）：
  - 采用 **glTF / GLB** 格式（若原模型不是此格式，后端启动时预转换）。
  - 启用 **Draco 几何压缩**、**KTX2 纹理压缩**。
  - 实现 **LOD（多细节层级）** 机制，大模型动态降面。
  - 加载时显示**进度条 + 百分比**。
- 场景基础组件：
  - 环境光 + 方向光 + 半球光，确保浮雕细节清晰可见。
  - 支持切换光照方向，便于观察不同角度的雕刻阴影。
  - 可选网格地面、背景色切换（黑/灰/白）。

### Step 1.3 — 3D 模式交互

- 基于 **OrbitControls**（Three.js 内置）实现：
  - **鼠标左键拖动** → 旋转模型。
  - **鼠标中键拖动** → 平移视角。
  - **滚轮** → 缩放。
- 额外功能：
  - 双击模型自动居中。
  - 右上角小导航指南针（Gizmo），显示当前朝向。
  - "重置视角"按钮。

### Step 1.4 — 2D 模式切换

- 顶部提供 **2D / 3D 切换按钮**。
- **进入 2D 模式时**：
  - 相机自动切换为**正交投影相机（OrthographicCamera）**。
  - 视角锁定为**正面视图**（即画面雕刻的一侧，通过模型的 bounding box + 预设法向量自动计算）。
  - 禁用旋转，仅保留：
    - 鼠标左键拖动 → 平移。
    - 滚轮 → 缩放。
- **首次加载画像石时**：可让用户手动标定"正面朝向"并保存到元数据中，后续自动使用该参数。
- **为后续标注模块预留接口**：2D 视图下暴露当前视图的投影矩阵，供 P3 标注模块使用。

### Step 1.5 — 多源资料面板

- 右侧面板顶部加入 **多视图切换标签**：
  - 3D 模型 / 正面拓片 / 线描图 / 原石照片（先预留接口，后续扩展）。
- 底部展示该画像石的**基础元数据卡片**：名称、编号、尺寸、年代、出土地等。

### Step 1.6 — 浏览模式右侧面板

浏览模式下右侧面板内容：
- **画像石基础信息**（从 MD 解析）。
- **分层结构化数据**（树状展示各层内容）。
- **光照 / 背景 / 显示选项**控制。
- **截图 / 录屏**工具（导出当前视角为图片或短视频，便于研究记录）。

# 🥈 第二部分：多石拼接模块（P2，第二优先级）

## 🎯 功能目标

支持用户将多块汉画像石加载至同一工作区，按"选边—吸附—微调—锁定"的结构化流程完成数字化拼接复原。这是本平台最具差异化价值的核心功能。

## ✅ 实现步骤

### Step 2.1 — 拼接工作区搭建

- 进入"拼接"模式后，左侧主视图切换为**拼接工作区**（新的 Three.js 场景，与浏览模块场景隔离但可复用模型加载器）。
- **最大支持同时加载 10 块**画像石模型。
- 右侧面板切换为**拼接控制面板**，包含：
  - 已加载画像石列表（缩略图 + 名称 + 锁定状态）。
  - "添加画像石"按钮（从 45 块库中选择，弹出缩略图选择器）。
  - 拼接操作区（见下文）。
  - "移除"按钮。

### Step 2.2 — 画像石边缘建模

每块画像石作为一个立方体（长方体）结构，具有 **6 个面**：
- `top`（上）、`bottom`（下）、`left`（左）、`right`（右）、`front`（正面 / 画面）、`back`（背面）

系统需为每块画像石**预计算 6 个面的 bounding plane**（通过模型 AABB 计算），用于后续吸附对齐。每个面存储：
- 中心点坐标
- 法向量
- 面的尺寸（宽 × 高）

### Step 2.3 — 选边拼接流程

**交互流程**：

1. 用户在右侧面板选择 **画像石 A**，在 3D 工作区点击它的某个面（如右面）→ 面高亮为**蓝色**。
2. 用户选择 **画像石 B**，点击它的对应面（如左面）→ 面高亮为**红色**。
3. 点击"**执行拼接**"按钮：
   - 系统自动计算位移矩阵，使 B 的选定面与 A 的选定面**完全贴合**。
   - B 的面法向量自动与 A 的面法向量**反向对齐**（贴合时两面法向量相反）。
   - B 被移动到正确位置。

**不允许纯自由拖拽**，必须通过选边方式保证拼接逻辑严谨。

### Step 2.4 — 微调操作（旋转 / 翻转 / 调头）

拼接完成后，若用户发现**边对上了但方向反了**（例如画面朝向不对、上下颠倒）：

1. **锁定参考块**：用户在列表中点击某块画像石的"🔒 锁定"，使其不动。
2. 对另一块（未锁定）执行微调：
   - **沿接触面法线旋转 90° / 180° / 270°**（快捷按钮）。
   - **水平翻转 / 垂直翻转**（快捷按钮）。
   - **自由微调旋转**（慢速滑块，用于细微偏差校正）。
   - **平移微调**（XYZ 方向 ± 按钮，步长可选 1mm / 5mm / 10mm）。
3. 每次操作实时预览，用户满意后点击"确认"。

### Step 2.5 — 组合锁定与链式拼接

1. 用户对当前两块的拼接效果满意 → 点击"**锁定组合**"。
2. 两块被合并为一个**逻辑组合体**，在列表中显示为一个展开项（可折叠）。
3. 继续拼接第三块时：
   - 用户选择**组合体上的某个面**（系统自动识别组合体的外侧可用面，内部已贴合的面不可选）。
   - 选择第三块的某个面。
   - 重复 Step 2.3 – Step 2.4 流程。
4. 依此类推，最多构建包含 10 块的拼接结构。

### Step 2.6 — 拼接方案保存与导出

- **保存拼接方案**：
  - 方案包含：参与的石块 ID 列表、每块的位姿矩阵（4×4）、锁定关系、创建时间、备注。
  - 存入数据库，用户可命名（如"武氏祠前石室西壁复原方案 v1"）。
- **方案管理面板**：
  - 列出所有已保存方案，支持加载、重命名、删除、复制。
- **导出**：
  - 导出为**合并的 GLB 模型**（整体复原件，可用于外部软件查看）。
  - 导出**拼接报告 PDF**（含截图 + 每块石料的信息 + 拼接顺序）。
  - 导出**位姿 JSON**（记录每块的 4×4 变换矩阵，便于其他研究者复现）。

### Step 2.7 — 拼接辅助功能

- **碰撞检测**：拼接时若两块发生非预期重叠，高亮警告。
- **缝隙测量**：吸附后若接缝存在缝隙（模型不规则），显示缝隙距离（mm）。
- **分解动画**：一键"爆炸视图"，各块沿原拼接方向分离，便于观察结构。
- **拼接步骤回放**：像 PPT 翻页一样，逐步展示用户的拼接过程，用于学术演示。

# 🥉 第三部分：图像标注模块（P3，最后开发）

## 🎯 功能目标

在浏览器与拼接器稳定后，扩展专业级、多模式、结构化、AI 辅助的标注系统，使标注成果可用于学术研究、知识图谱构建与 AI 训练。

> ⚠️ 本模块在 P1 / P2 完成并稳定后再启动开发。
> ⚠️ **开发前必须读取 `E:\WSC3D\参考图` 目录下的 3 张示例图**，严格参考其 UI 风格、功能布局与交互逻辑。

## ✅ 实现步骤

### Step 3.1 — 标注画布与坐标系

- **触发条件**：仅在 **2D 模式**下可创建标注。
- 在 Three.js 渲染层之上**叠加一层 Canvas 2D 画布**（推荐使用 **Konva.js** 或 **Fabric.js**）。
- **坐标映射**：
  - 标注存储时转换为**模型世界坐标**（基于相机投影矩阵反算），而非屏幕像素坐标。
  - 这样标注可以**在 3D 模式下也正确显示**（投影到模型表面）。
- 3D 模式下，标注以**贴合模型表面的叠加层**或**标注点气泡**的形式呈现（参考示例图 2 的气泡样式）。

### Step 3.2 — 多模式标注工具（左侧工具栏）

> 参考示例图 1：左侧图标工具栏 + 主画布 + 右侧三个工具演示区（SAM 标注、矩形标注、钢笔标注）。

实现以下标注方式：

| 工具 | 实现要点 |
|------|---------|
| **矩形框选** | 拖拽绘制，支持调整四角大小 |
| **圆形 / 椭圆框选** | 拖拽绘制椭圆，支持半径与旋转调整 |
| **钢笔（多边形 / 贝塞尔曲线）** | 点击打点形成多边形，双击闭合；支持贝塞尔控制柄精细调整曲线 |
| **SAM 智能标注**（亮点功能，参考示例图 1） | 集成 Meta 的 Segment Anything Model，单击一个点即可自动分割出图像物体轮廓 |
| **选择 / 移动** | 点选已有标注、拖动、删除 |
| **撤销 / 重做** | 基于操作栈实现完整 undo/redo |

**SAM 集成方案**：
- 后端部署 SAM 推理服务（Python + PyTorch，推荐使用轻量化的 **MobileSAM** 或 **FastSAM** 提高响应速度）。
- 前端将当前 2D 视图截图 + 用户点击坐标发送给后端，返回 mask 多边形轮廓。
- 前端将返回的轮廓以可编辑多边形形式呈现，用户可微调。

### Step 3.3 — 标注属性填写（右侧面板）

> 参考示例图 2：右侧面板为 Tab 结构（对象 / 术语表 / 我的标注）。

用户完成几何绘制后，右侧面板进入**标注编辑态**，分为三个 Tab：

#### Tab 1：结构化数据快选（对接 MD 文档）
- 加载当前画像石的分层结构化数据（复用 P1 的解析结果）。
- **树状展示**：第一层 → 左起第一幅（标题摘要）→ 详细描述。
- 用户**点击某个条目**，即可将其内容自动填入当前标注，无需手动输入。

#### Tab 2：受控术语词表（参考示例图 2 右侧词库列表）
- 建立统一的**汉画像石术语库**（JSON 文件，可后台维护）。
- 分类示例：
  - 人物类：羽人、使者、侍者、西王母、东王公、孔子、乐工……
  - 动物类：龙、凤、朱雀、玄武、白虎、青龙、鱼、鸟……
  - 器物类：马车、鼎、博山炉、乐器、兵器……
  - 场景类：宴饮、车骑出行、狩猎、神仙会、历史故事……
  - 纹饰类：云气纹、卷草纹、几何纹、菱形纹……
- 用户从词表中**直接勾选**，避免自由输入导致的标签混乱。
- 支持搜索、多选、自定义新增。

#### Tab 3：我的标注（列表管理）
- 展示当前画像石的所有标注条目。
- 点击某条标注 → 高亮对应几何区域并居中显示。
- 支持批量删除、导出、过滤。

### Step 3.4 — 知识图谱构建（半自动，参考示例图 3）

**目标**：当标注积累到一定量后，自动生成该画像石及整个数据集的**语义关系图谱**，UI 呈现参考示例图 3 的节点-边可视化风格。

- **关系建模**：
  - 实体节点：每个标注对象（人物、动物、器物、场景）。
  - 边关系：
    - **空间关系**（自动推导）：A 在 B 左侧 / A 骑在 B 身上 / A 包含 B。
    - **语义关系**（半自动 + 人工）：西王母 —有—→ 玉兔、羽人 —守护—→ 西王母。
- **可视化**：
  - 右侧面板提供"知识图谱"Tab，使用 **D3.js** / **Cytoscape.js** / **ECharts Graph** 呈现。
  - 节点可点击，点击后左侧画布高亮对应标注。
  - 节点样式参考示例图 3 的橙色圆形节点 + 中文标签。
- **半自动推荐**：
  - 基于已有标注自动推荐关联术语（例：标注了"西王母"，推荐"玉兔""三足乌""九尾狐"）。
  - 推荐规则：先用**共现统计 + 规则库**实现，后期可升级为图神经网络。

### Step 3.5 — 标注数据存储与版本管理

- **后端数据库**：推荐 **SQLite**（本地开发）或 **PostgreSQL**（部署）。
- **标注数据表结构**：
  ```
  annotations (
    id, stone_id, type(rect/circle/polygon/sam),
    geometry_json, world_coords_json,
    label_text, term_ids[], linked_md_node,
    created_at, updated_at, author, version
  )
  ```
- **类 Git 版本管理**（参考示例图 2 的"类 git 版本管理"说明）：
  - 每次保存形成一个 **快照（commit）**，带作者 + 时间戳 + 备注。
  - 支持**查看历史版本、版本对比、回滚到指定版本**。
  - 实现思路：标注变更存储为 **JSON Patch** 增量，结合全量快照定期合并。
